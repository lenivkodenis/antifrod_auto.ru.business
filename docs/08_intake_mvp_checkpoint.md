# Контроль входа автомобиля — MVP checkpoint

Дата фиксации: 2026-05-18

## Статус

Блок **«Контроль автомобиля на входе»** считается собранным на уровне MVP-методологии.

Это не финальный продукт и не готовый промышленный модуль, но уже есть рабочая логика:

- какие данные нужны;
- какие нарушения процесса искать;
- какие экономико-рыночные риски считать;
- как формулировать вывод для собственника;
- как собрать первый отчет по Excel-выгрузке Авто.ру Бизнес;
- как в будущем перенести эту механику на API.

## Почему пока MVP строится на Excel

Стратегически предпочтительный формат будущего продукта — API-интеграция с Авто.ру Бизнес и выдача отчета собственнику через Telegram-бота, HTML-отчет или PDF.

Но на текущем этапе доступность API и конкретных полей еще не подтверждена.

Поэтому MVP строится на ручной Excel-выгрузке, чтобы быстро проверить продуктовую ценность и методологию.

Целевая будущая логика:

> Собственник не скачивает Excel. Он открывает Telegram, вызывает бота и получает короткий отчет по бизнесу на телефоне.

Excel сейчас используется только как прототипный источник данных.

## Текущая продуктовая рамка

Проект не ограничивается выкупом. Он покрывает контроль жизненного цикла автомобиля с пробегом у дилера.

Три больших блока продукта:

1. **Вход автомобиля** — оценка, осмотр, расчет экономики, согласование, прием на склад.
2. **Управление автомобилем на стоке** — публикация, продвижение, цена, переоценка, спрос, срок хранения, GM2.
3. **Выход / продажа** — скидка, продажа ниже рынка, продажа без публикации, итоговая маржа, отклонение от плана.

Типы контрактов учитываются как контекст:

- trade-in;
- trade-up;
- выкуп;
- комиссия;
- ком-трейд.

Но основная структура контроля строится не по типам договоров, а по этапам жизненного цикла автомобиля.

## Методологические решения по входу автомобиля

### 1. Диагностика не является must-have полем Авто.ру Бизнес

Диагностика часто проводится в системе дилера или сервисном контуре и может прикладываться позже файлами к карточке автомобиля.

Отсутствие диагностики в выгрузке Авто.ру Бизнес не доказывает, что диагностика не проводилась.

### 2. Мобильный осмотр — ключевой цифровой след

Для контроля входа главным доказательным процессом является мобильный осмотр.

Формулировка:

> Автомобиль попал на склад без завершенного мобильного осмотра в Авто.ру Бизнес. Это не доказывает, что его физически не смотрели, но в системе нет завершенного цифрового следа осмотра.

### 3. Ручной расчет экономики определяется по округлению трех полей

Ключевые поля:

- `Собственнику`;
- `Маржа, ₽`;
- `Цена продажи`.

Если хотя бы одно из трех полей не кратно 1 000 ₽, экономика выглядит как автоматический / неотрегулированный вручную расчет.

Пример ручной оценки:

- собственнику: 500 000 ₽;
- маржа: 100 000 ₽;
- цена продажи: 600 000 ₽.

Пример автоматического расчета:

- собственнику: 583 271 ₽;
- маржа: 117 842 ₽;
- цена продажи: 701 113 ₽.

### 4. GM2 важнее простой маржи

GM1 или плановая маржа не должны быть единственным ориентиром. Если в данных есть GM2, его нужно анализировать отдельно.

### 5. Оценка должна иметь управленческий след

Для принятого автомобиля важны:

- завершенная оценка;
- завершенный мобильный осмотр;
- ручной расчет экономики;
- согласование при риске;
- прогноз срока продажи;
- ценовой коридор;
- связь с рынком и ОЦП.

## Основные процессные сигналы MVP

Для автомобилей, принятых на склад, проверяются:

1. Экономика не выглядит рассчитанной вручную.
2. Мобильный осмотр не завершен.
3. Оценка не завершена.
4. Оценка не согласована.

## Основные экономико-рыночные сигналы MVP

Проверяются:

1. Цена продажи выше 100% к рынку.
2. Маржа ниже 10%.
3. Выкуп выше ОЦП.
4. Выкуп + подготовка выше ОЦП.
5. Цена продажи ниже минимального прогноза.
6. Цена продажи ниже ОЦП.
7. Не задан прогноз срока продажи.
8. Нет минимальной/максимальной рекомендованной цены.
9. Мало похожих автомобилей для рыночного расчета.
10. Подготовка пустая или равна 0.
11. Не заполнен канал.
12. Не заполнено состояние.

## Агрегированные результаты по первому тестовому файлу

NDA: сохраняются только агрегированные значения, без VIN, ФИО, ссылок на оценки, конкретных автомобилей и сделок.

База:

- всего строк в выгрузке: **4 039 оценок**;
- строк `На складе = Да`: **522**;
- уникальных автомобилей на складе: **495**.

# Контроль автомобиля на стоке во время продажи

Основные темы для следующего этапа:

- автомобиль получает лиды, но не продается.

Для следующего этапа желательно использовать выгрузку **«Актуальный склад»** из Авто.ру Бизнес.
