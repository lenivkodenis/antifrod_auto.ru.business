## Экран 5. Приоритеты проверки

### Приоритет 1. Критичные сделки

Проверить автомобили, где одновременно есть несколько признаков:

- экономика не рассчитана вручную;
- мобильный осмотр не завершен;
- оценка не завершена;
- оценка не согласована;
- маржа ниже 10%;
- выкуп выше ОЦП;
- цена продажи ниже минимального прогноза.

### Приоритет 2. Процессные нарушения

Проверить, почему в Авто.ру Бизнес массово не закрываются:

- согласование цены;
- мобильный осмотр;
- статус оценки;
- прогноз срока продажи;
- ценовой коридор.

### Приоритет 3. Настройки методологии

Проверить, что именно считается обязательным для дилера:

- когда нужен мобильный осмотр;
- кто обязан согласовывать цену;
- какой минимальный порог маржи допустим;
- какой целевой GM2 должен быть на входе;
- когда разрешен выкуп выше ОЦП;
- как должен заполняться срок продажи.

## Экран 6. Управленческие действия

| Что сделать | Зачем |
|---|---|
| Ввести обязательное согласование рискованных оценок | Чтобы собственник видел, кто разрешил вход автомобиля |
| Запретить постановку на склад без завершенной оценки | Чтобы автомобиль не входил в бизнес до закрытия расчета |
| Сделать мобильный осмотр ключевым цифровым следом | Чтобы отделить реальный осмотр от формального приема |
| Добавить проверку округления денежных полей | Чтобы ловить оценки без ручного расчета экономики |
| Ввести обязательный прогноз срока продажи | Чтобы оценивать ликвидность и влияние на GM2 |
| Контролировать выкуп выше ОЦП | Чтобы не принимать автомобили с убытком, заложенным на входе |
| Ввести отчет по сотрудникам и источникам | Чтобы находить системные отклонения, а не разовые ошибки |
