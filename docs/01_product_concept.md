# Концепция продукта

## Рабочее название

**Система контроля качества сделок с автомобилями с пробегом**

Дополнительные варианты позиционирования:

- контроль качества стока с пробегом;
- антифрод и контроль маржи автомобилей с пробегом;
- контроль входа, стока и выхода автомобиля;
- управленческий контур автомобилей с пробегом.

## Для кого продукт

Основные пользователи:

- собственник дилерского бизнеса;
- генеральный директор;
- руководитель направления автомобилей с пробегом;
- руководитель отдела продаж;
- внутренний аудитор / контролер;
- команда Авто.ру Бизнес как поставщик управленческой аналитики.

## Главная боль

В направлении автомобилей с пробегом много решений принимается на доверии к конкретным людям: байеру, оценщику, руководителю, продавцу.

Ошибки и злоупотребления могут возникать на трех этапах:

1. На входе автомобиля — приняли дорого, без осмотра, без расчета экономики, без согласования.
2. Во время нахождения на стоке — не переоценили, не продвинули, не опубликовали, не управляли GM2.
3. На выходе — дали необоснованную скидку, продали ниже рынка, продали без публикации, разрушили маржу.

## Продуктовая идея

Система должна не заменять руководителя, а давать ему короткие и регулярные сигналы:

- что произошло за день;
- что накопилось за неделю;
- где системные проблемы за месяц;
- какие автомобили требуют внимания;
- какие сотрудники, источники или процессы чаще связаны с отклонениями;
- где потенциально теряется маржа.

## Форматы продукта

### 1. Консалтинговый продукт

Аудит текущего процесса дилера:

- выгрузка данных;
- поиск отклонений;
- интервью с руководителями;
- карта рисков;
- рекомендации;
- отчет для собственника.

### 2. Методология и регламенты

Комплект правил для дилера:

- как должна проходить оценка;
- какие поля обязательны;
- когда требуется согласование;
- как контролировать GM2;
- как управлять переоценкой;
- как проверять продажу.

### 3. Управленческий отчет

Регулярный отчет для собственника:

- ежедневный короткий срез;
- еженедельный отчет по отклонениям;
- ежемесячная карта качества процесса.

### 4. Будущий продуктовый слой Авто.ру Бизнес

Не обязательно разрабатывать плагин на этом этапе. Но можно проектировать будущую продуктовую логику:

- мобильное уведомление;
- карточка отклонения;
- рейтинг критичности;
- список автомобилей в зоне риска;
- отчет по сотрудникам;
- отчет по источникам;
- отчет по потере маржи.

## Три блока контроля

### Блок 1. Вход автомобиля

Главный вопрос:

> Правильно ли автомобиль попал в контур дилера?

Контрольные темы:

- мобильный осмотр;
- завершенность оценки;
- индивидуальность экономического расчета;
- срок продажи;
- прогноз цены продажи;
- ценовой коридор;
- маржа;
- GM2;
- согласование отклонений.

### Блок 2. Управление автомобилем на стоке

Главный вопрос:

> Правильно ли автомобилем управляют, пока он продается?

Контрольные темы:

- публикация;
- фотографии и контент;
- цена к рынку;
- продвижение;
- переоценка;
- срок хранения;
- лиды, звонки, просмотры;
- GM2;
- план действий по возрастному стоку.

### Блок 3. Выход / продажа

Главный вопрос:

> Правильно ли автомобиль продали и не разрушили ли экономику сделки?

Контрольные темы:

- продажа ниже рынка;
- необоснованная скидка;
- продажа без публикации;
- продажа ниже минимальной расчетной цены;
- фактическая GM2;
- отличие факта от плана на входе;
- согласование скидок;
- подозрительные повторяющиеся паттерны.

## Принцип продукта

Система должна искать не виноватых, а отклонения.

Цель — дать руководителю действие:

- проверить конкретную машину;
- запросить объяснение;
- изменить регламент;
- ограничить самостоятельное принятие решений;
- пересмотреть мотивацию;
- усилить контроль на слабом этапе.
