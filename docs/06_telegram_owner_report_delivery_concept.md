# Концепция доставки отчета собственнику через Telegram-бота

## Идея

В будущем собственник или генеральный директор должен иметь возможность запросить отчет по контролю автомобилей с пробегом через Telegram-бота.

Бот не должен быть просто каналом отправки файла. Он должен быть управленческим интерфейсом: коротко показать главный риск, дать возможность провалиться в детали и получить отчет в нужном формате.

## Основной сценарий

Пользователь пишет боту или нажимает кнопку:

> Отчет по входу автомобилей

Бот возвращает короткую сводку:

- сколько автомобилей принято за период;
- сколько вошло с нарушениями процесса;
- сколько имеют экономико-рыночные риски;
- какие сигналы критичные;
- что проверить в первую очередь.

После этого бот предлагает действия:

- открыть интерактивный отчет;
- скачать PDF;
- показать критичные автомобили;
- показать отчет по сотрудникам;
- показать отчет по источникам;
- изменить период;
- изменить дилерский центр / группу дилеров;
- изменить пороги риска.

## Возможные форматы выдачи

### 1. Короткое Telegram-уведомление

Формат для ежедневного или оперативного контроля.

Пример:

> Контроль входа автомобилей  
> За период принято 495 авто.  
> 485 — без завершенного мобильного осмотра.  
> 387 — экономика выглядит не рассчитанной вручную.  
> 56 — имеют экономико-рыночный риск.  
> Приоритет: проверить автомобили с маржой ниже 10% и выкупом выше ОЦП.

Плюсы:

- быстро читается;
- удобно для ежедневного контроля;
- не требует открытия файла.

Минусы:

- мало деталей;
- неудобно анализировать автомобили, сотрудников и источники.

### 2. PDF-отчет

Формат для собственника, генерального директора или совещания.

Содержание:

- главный вывод;
- сводка нарушений;
- экономико-рыночные риски;
- топ критичных сигналов;
- динамика за период;
- сотрудники / источники / дилерские центры;
- рекомендации.

Плюсы:

- удобно пересылать;
- выглядит как законченный управленческий документ;
- подходит для регулярного отчета.

Минусы:

- статичный;
- фильтры и детализация ограничены.

### 3. HTML-отчет / лендинг

Формат для интерактивного анализа.

Возможности:

- фильтр по периоду;
- фильтр по дилерскому центру;
- фильтр по сотруднику;
- фильтр по типу контракта;
- фильтр по критичности;
- список автомобилей в зоне риска;
- раскрытие карточки сигнала;
- переход от агрегата к конкретной сделке;
- экспорт в PDF.

Плюсы:

- самый продуктовый формат;
- можно делать кликабельные сценарии;
- подходит для будущего модуля Авто.ру Бизнес.

Минусы:

- требует разработки;
- нужно решать безопасность доступа и хранение данных.

### 4. Гибридный формат

Оптимальная логика для MVP:

1. Telegram присылает короткую сводку.
2. В сообщении есть кнопки:
   - `Открыть отчет`;
   - `Скачать PDF`;
   - `Показать критичные`;
   - `Изменить период`.
3. Полный отчет открывается как HTML-страница.
4. PDF формируется как статичная версия HTML-отчета.

## Логика периодов

Бот должен поддерживать несколько периодов:

- сегодня;
- вчера;
- неделя;
- месяц;
- произвольный период;
- последний загруженный отчет.

Для собственника ключевые режимы:

- ежедневный контроль — короткие критичные сигналы;
- еженедельный контроль — процессные и экономические отклонения;
- ежемесячный контроль — динамика, сотрудники, источники, деньги.

## Блоки будущего Telegram-бота

### Блок 1. Вход автомобилей

Сигналы:

- экономика не рассчитана вручную;
- мобильный осмотр не завершен;
- оценка не завершена;
- оценка не согласована;
- маржа ниже порога;
- выкуп выше ОЦП;
- цена продажи выше рынка;
- цена продажи ниже минимального прогноза.

### Блок 2. Управление на стоке

Будущие сигналы:

- автомобиль не опубликован;
- проблемы размещения;
- нет фотографий;
- нет продвижения;
- цена выше рынка;
- цена не менялась дольше норматива;
- срок склада выше нормы;
- GM2 ниже порога;
- нет лидов / звонков / просмотров.

### Блок 3. Выход / продажа

Будущие сигналы:

- продажа ниже рынка;
- продажа ниже минимальной цены;
- продажа без публикации;
- скидка выше порога;
- GM2 ниже порога;
- факт хуже плана на входе;
- подозрительные повторяющиеся паттерны по сотрудникам или покупателям.

## Рекомендуемый MVP

Первый MVP не должен начинаться с полноценного приложения.

Оптимальный путь:

1. Excel-выгрузка Авто.ру Бизнес.
2. Скрипт анализа.
3. Генерация JSON с сигналами.
4. Генерация HTML-отчета.
5. Генерация PDF-версии.
6. Telegram-бот отправляет короткую сводку и ссылку на HTML/PDF.

## Принцип безопасности

Нельзя отправлять в Telegram чувствительные данные без контроля доступа.

Для MVP лучше:

- показывать в Telegram только агрегаты;
- детальный отчет открывать по защищенной ссылке;
- не хранить VIN и ФИО в публичных отчетах;
- использовать обезличенные идентификаторы автомобилей;
- ограничивать срок жизни ссылки;
- логировать, кто открыл отчет.

## Продуктовый вывод

Telegram-бот должен стать не просто способом доставки отчета, а интерфейсом управленческого контроля.

Главная логика:

> Собственник не ищет проблему в таблицах. Он вызывает бота и получает список отклонений, приоритетов и действий.
