## NDA-принцип

В файл сохранены только агрегированные показатели. Не сохранены VIN, госномера, ФИО сотрудников, телефоны, ссылки на оценки, конкретные автомобили или коммерческие условия конкретных сделок.

## Объем данных

- Всего строк в выгрузке: **4 039 оценок**.
- Строк со статусом `На складе = Да`: **522**.
- Уникальных автомобилей на складе по полю `# авто на складе`: **495**.

Для управленческого отчета собственнику правильнее считать уникальные автомобили по `# авто на складе`, потому что по одному автомобилю может быть несколько строк оценки.

## Проверяемые нарушения процесса входа

Для принятого автомобиля проверяются четыре сигнала:

1. **Экономика не просчитана вручную** — хотя бы одно из трех полей `Собственнику`, `Маржа, ₽`, `Цена продажи` не кратно 1 000 ₽, либо экономика не заполнена.
2. **Мобильный осмотр не завершен** — статус мобильного осмотра не равен `Завершен`.
3. **Оценка не завершена** — статус оценки не равен `Завершена`.
4. **Оценка не согласована** — пусты поля `Согласованная цена` и/или `Цену согласовал`.

## Управленческая формулировка

За отчетный период в выгрузке найдено **495 уникальных автомобилей**, принятых на склад. По всем 495 автомобилям есть минимум один признак несоблюдения процесса входа.

Самые массовые отклонения:

- нет зафиксированного согласования оценки — 495 автомобилей;
- мобильный осмотр не завершен — 488 автомобилей;
- экономика не выглядит просчитанной вручную — 389 автомобилей;
- оценка не завершена — 364 автомобиля.

## Методологические ограничения

1. Отсутствие завершенного мобильного осмотра в Авто.ру Бизнес не доказывает, что автомобиль физически не осматривали. Это означает, что в системе нет завершенного цифрового следа осмотра.
2. Неокругленные денежные поля являются сильным признаком автоматического или формального расчета, но итоговое управленческое заключение лучше усиливать другими сигналами.
3. Отсутствие согласования в выгрузке означает, что в полях Авто.ру Бизнес нет следа согласованной цены и согласующего лица. Если дилер согласует сделки в другой системе, это нужно учитывать отдельно.
4. Для отчета собственнику нужно считать уникальные автомобили, а не строки оценки, потому что по одному автомобилю может быть несколько оценок.
