# Антифрод и контроль качества автомобилей с пробегом

Рабочий репозиторий проекта по созданию системы контроля качества сделок с автомобилями с пробегом для дилеров на базе логики и данных Авто.ру Бизнес.

## Цель проекта

Создать продуктовую и методологическую систему, которая помогает собственнику, генеральному директору или руководителю направления автомобилей с пробегом видеть управленческие отклонения на всем жизненном цикле автомобиля:

1. **Вход автомобиля** — оценка, осмотр, расчет экономики, согласование, прием в контур дилера.
2. **Управление на стоке** — публикация, цена, продвижение, переоценка, спрос, срок хранения, GM2.
3. **Выход / продажа** — скидки, продажа ниже рынка, продажа без публикации, итоговая маржа, отклонение от плановой экономики.

Проект не сводится только к выкупу. В фокусе все основные типы сделок с автомобилями с пробегом:

- trade-in;
- trade-up;
- выкуп;
- комиссия;
- ком-трейд.

Риски по этим типам контрактов в основном похожи, поэтому основная структура проекта строится не по договорам, а по трем этапам жизненного цикла автомобиля.

## Главная продуктовая идея

Собственник не должен вручную проверять каждую оценку, каждую машину на складе и каждую продажу. Система должна находить отклонения от нормального процесса и показывать их в понятном управленческом виде.

Не формулируем вывод так:

> Сотрудник виноват.

Формулируем так:

> Сделка прошла с отклонением: нет завершенного мобильного осмотра, экономика похожа на шаблонную, оценка не закрыта до попадания автомобиля на склад, GM2 ниже целевого уровня.

## Основные артефакты

```text
docs/       — методология продукта
rules/      — правила контроля и риск-сигналы
schemas/    — описание выгрузок и словарь данных
prompts/    — промпты для Codex/ChatGPT
.agents/    — skill для агента
scripts/    — будущие скрипты анализа Excel-выгрузок
samples/    — обезличенные примеры данных
outputs/    — примеры отчетов
```

## Важное ограничение по данным

Репозиторий не должен содержать сырые клиентские выгрузки, реальные VIN, ФИО сотрудников, коммерческие условия сделок, токены, cookies, пароли или скриншоты с чувствительной информацией.

Для разработки допускаются только:

- обезличенные примеры;
- синтетические тестовые данные;
- структура колонок;
- правила контроля;
- методология;
- код анализа.

## Текущий фокус

Первая часть проекта: **контроль входа автомобиля**.

Ключевые темы:

- был ли завершенный мобильный осмотр;
- не попал ли автомобиль на склад до завершения оценки;
- не оставлена ли экономика в шаблонном состоянии;
- изменял ли оценщик параметры расчета;
- выбран ли срок продажи;
- есть ли управленческий ценовой коридор;
- не выглядит ли входная цена завышенной;
- есть ли признаки слабого контроля или потенциального коррупционного риска.
