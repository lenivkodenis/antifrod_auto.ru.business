# Skill: Auto.ru Used Cars Control

## Назначение

Используй этот skill для задач проекта по контролю качества сделок с автомобилями с пробегом у дилеров на базе логики Авто.ру Бизнес.

Проект должен помогать собственнику, генеральному директору и руководителю направления автомобилей с пробегом видеть отклонения в процессе и потенциальные коррупционные/экономические риски.

## Основная рамка

Не раскладывай риски по типам контрактов как основную структуру. Типы контрактов учитывать как контекст:

- trade-in;
- trade-up;
- выкуп;
- комиссия;
- ком-трейд.

Основная структура всегда состоит из трех блоков:

1. Вход автомобиля.
2. Управление автомобилем на стоке.
3. Выход / продажа.

## Текущий фокус

Текущий фокус проекта — блок **вход автомобиля**.

Вход автомобиля включает:

- оценку;
- мобильный осмотр;
- экономический расчет;
- прогноз срока продажи;
- прогноз цены продажи;
- ценовой коридор;
- GM2;
- согласование;
- попадание автомобиля на склад.

## Важное правило по диагностике

Не делай диагностику must-have полем Авто.ру Бизнес.

Диагностика может проводиться во внутренней системе дилера и прикладываться позже файлами к карточке автомобиля. Поэтому отсутствие диагностики в выгрузке Авто.ру Бизнес нельзя трактовать как доказательство отсутствия диагностики.

Главный цифровой след осмотра в Авто.ру Бизнес — **мобильный осмотр**.

## Важное правило по экономике

Авто.ру Бизнес может предлагать экономику по умолчанию. Если оценщик ничего не менял, дефолтная экономика может сохраниться.

Ищи признаки шаблонной экономики:

- типовая маржа: 0%, 10%, 20%;
- типовая подготовка;
- не выбран срок продажи;
- не настроена прогнозная цена продажи;
- не заполнен ценовой коридор;
- нет признаков индивидуального расчета под конкретный автомобиль.

Не утверждай жестко: «оценщик ничего не считал», если нет истории изменений.

Формулируй так:

> Экономика оценки выглядит шаблонной. Не видно признаков индивидуальной настройки расчета.

## GM2

GM2 важнее простой плановой маржи. Если в данных есть GM2, обязательно анализируй:

- GM2 в рублях;
- GM2 в процентах;
- GM2 ниже целевого порога;
- случаи, где GM1 выглядит нормально, но GM2 слабая или отрицательная.

## Формулировки

Не обвиняй сотрудников без доказательств.

Используй язык управленческих сигналов:

- «требует проверки»;
- «есть признак слабого контроля»;
- «не видно цифрового следа»;
- «экономика выглядит шаблонной»;
- «сделка прошла с отклонением от нормального процесса».

## Типовой результат анализа

Результат должен содержать:

- количество автомобилей / оценок в анализе;
- период данных;
- количество автомобилей на складе;
- ключевые сигналы;
- критичность;
- ограничения интерпретации;
- управленческие действия.

## Конфиденциальность

Не сохраняй и не публикуй:

- сырые Excel-файлы клиентов;
- VIN;
- ФИО сотрудников;
- реальные коммерческие условия сделок;
- токены, пароли, cookies;
- скриншоты с чувствительными данными.

В репозитории используй только методологию, правила, схемы, обезличенные примеры и код.
